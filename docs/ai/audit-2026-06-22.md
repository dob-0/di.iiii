## Phase 3 — `scripts/e2e-smoke.mjs`

Result: 21 passed, 16 failed. All 16 failures traced to causes outside the app:

1. **Stale test fixture** — the script hardcodes `PROJECT_ID = 'default-scene-test'`, which does not exist in this DB's `main` space (confirmed via `/api/spaces/main/projects` — real projects are `test-file-project`, `verify-group-test`, etc.). Directly explains the `Backend .../document → 404` failure.
2. **Stale UI selectors** — manually opening a real project in Studio shows actual tab/button labels (`World▾`, `Navigate`, `Edit`, `Library`, `Assets`, `Files`, `Inspector`, `Structure`, `Present`, `Publish`, `Activity`) — there is no button literally named "View" or "Graph" anymore. The script's selectors predate a UI copy change.
3. **Expected Beta behavior** — opening Beta on an existing legacy project correctly shows only `"Cursor is material. Double-click to place Node 0."`, matching documented behavior (Beta topbar/tabs/outliner stay hidden until Node 0 is manually placed). The script expects those elements present without that step.

**Recommendation:** `scripts/e2e-smoke.mjs` needs its fixture project ID and Studio tab selectors updated, and its Beta-editor assertions reworked to either place Node 0 first or target a Beta-native (not legacy-imported) project. Filing this as a script-maintenance task, not an app bug.
